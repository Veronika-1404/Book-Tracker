## Описание

*Book Tracker — это десктопное приложение с графическим интерфейсом для учёта прочитанных книг. Программа позволяет добавлять книги с указанием названия, автора, жанра и количества страниц, просматривать их в виде таблицы, фильтровать по жанру и объёму, а также сохранять данные в JSON‑файл с автоматической фиксацией изменений в локальном Git‑репозитории.*

**Программа решает следующие задачи:**
* систематизация списка прочитанных книг; 
* быстрый поиск книг по жанру или количеству страниц; 
* надёжное хранение данных с возможностью восстановления; 
* отслеживание изменений через Git.

## Требования для запуска

Что нужно установить на компьютер: 
* Python 3.6 или выше 
* Библиотеки: `PySimpleGUI`:`pip install PySimpleGUI`;
  
## Как запустить

```bash 
git clone https://github.com/Veronika-1404/Book-Tracker
cd Book-Tracker
python main.py
```
## Как пользоваться

1.  При запуске открывается окно с полями ввода и таблицей:
    -   **Название книги** — введите название книги.
    -   **Автор** — укажите автора.
    -   **Жанр** — укажите жанр книги.
    -   **Количество страниц** — введите число страниц.
2.  Заполните поля и нажмите кнопку **«Добавить книгу»** — книга появится в таблице внизу окна.
3.  Для фильтрации:
    -   введите часть названия жанра в поле **«Фильтр по жанру»**;
    -   укажите минимальное количество страниц в поле **«Страницы >»**;
    -   таблица автоматически обновится.
4.  Нажмите **«Сохранить»**, чтобы записать данные в `books.json` и закоммитить их в Git.  
5.  Нажмите **«Загрузить»**, чтобы восстановить данные из `books.json`.
  
## Пример работы

Сценарий 1. **Добавление книги:**
-   Название книги: «1984»
-   Автор: «Джордж Оруэлл»
-   Жанр: «Антиутопия»
-   Количество страниц: `328`
-   Нажать **«Добавить книгу»** → книга появляется в таблице, поля очищаются.

Сценарий 2. **Фильтрация:**
-   В поле **«Фильтр по жанру»** ввести: `анти`
-   В поле **«Страницы >»** ввести: `200`
-   Результат: в таблице отображается только книга «1984».

Сценарий 3. **Сохранение и загрузка:**
-   Добавить 2–3 книги.
-   Нажать **«Сохранить»** → появляется сообщение «Данные сохранены в books.json и закоммичены в Git!».
-   Закрыть приложение, перезапустить, нажать **«Загрузить»** → книги восстанавливаются из `books.json`.

Сценарий 4. **Проверка валидации:**

-   Оставить поле **«Название книги»** пустым и нажать **«Добавить книгу»** → сообщение «Все поля, кроме фильтра, должны быть заполнены!».
-   Ввести текст в поле **«Количество страниц»** (например, «сто») → сообщение «Количество страниц должно быть положительным числом!».

## Тестирование

| Тип теста  | Пример ввода | Результат |
|-----|----------|---------------------|
| Позитивный | Корректные данные (название, автор, жанр, число страниц) | ✅ Книга добавлена в таблицу |
| Негативный | Пустое поле «Название книги» | ❌ Ошибка: «Все поля, кроме фильтра, должны быть заполнены!» |
| Негативный | Текст в поле «Количество страниц» (например, «сто») | ❌ Ошибка: «Количество страниц должно быть положительным числом!» |
| Граничный | Количество страниц = `1` | ✅ Книга добавлена |
| Фильтрация | Жанр: `роман`, страницы > `300` | ✅ Отображаются книги, соответствующие критериям |
| Сохранение | Нажать «Сохранить» после добавления книги | ✅ Данные записаны в `books.json`, коммит в Git |
| Загрузка | Нажать «Загрузить» после перезапуска приложения | ✅ Данные восстановлены из `books.json` |
